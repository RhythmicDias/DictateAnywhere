# Fix All 26 DictateAnywhere Issues

Fix every issue catalogued in [ISSUES.md](file:///d:/PythonProjects/DictateAnywhere/ISSUES.md), grouped by file/component to minimise context switching. No feature additions — strictly bug fixes and code hygiene.

## User Review Required

> [!IMPORTANT]
> **I-01 (failing test):** The test asserts `clean_whisper_artifacts("Thank you.") == ""` but the function's own docstring says *"Do NOT strip real words."* I propose **adding a short hallucination-phrase list** (`"Thank you."`, `"Thanks for watching."`, `"You."`) rather than deleting the test, since Whisper genuinely hallucinates these on silence. Please confirm this approach.

> [!IMPORTANT]
> **I-14 (punctuation spacing):** The fix adds a post-processing regex that collapses ` .` → `.`, ` ,` → `,` etc. This will change existing transcription output — spaces before punctuation will no longer appear. Confirm this is desired.

> [!WARNING]
> **I-06 (Azure double-encoding):** Changing from WAV→raw PCM in the push stream is a functional fix for Azure users. If you don't currently use Azure cloud, this is low-risk. If you do, it should be tested with a real key.

## Open Questions

> [!IMPORTANT]
> **I-16 (modal settings):** Removing `grab_set()` makes settings non-modal. Users could start dictation while settings are open. Is that acceptable, or should I keep modality and just allow the floating widget to stay interactive?
